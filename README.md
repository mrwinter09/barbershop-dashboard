## ğŸ“˜ OpenAPI Integration (Users API)

This project includes an **OpenAPI-first structure** for the `/users` endpoint:

- `openapi.yaml` defines the schema and operations.
- `openapi-typescript` generates strongly-typed interfaces into `src/lib/schema.d.ts`.
- React Query hooks (`useUsers`, `useCreateUser`) are implemented manually using those generated types.

### Why manual hooks instead of `openapi-react-query`?

For this assignment, I chose to:

- Use **`openapi-typescript`** for type generation.
- Write my own **typed React Query hooks** for fetching and mutations.

This keeps the architecture:

- **Lightweight and explicit** â€“ no hidden generated client logic.
- **Type-safe** â€“ all request/response contracts come from the OpenAPI spec.

If needed, this setup can later be extended to use `openapi-react-query` for autogenerated hooks, without changing the overall structure.
